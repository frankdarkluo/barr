## 一句话总结

到这一阶段，项目的重点已经从“还有没有更强 timing”收束到“把 protocol 清干净”：

- strict Phase C 证明了 intervention 本身有效；
- `k=2` 仍然是当前最好的 first candidate，但优势是温和的；
- sample-grouped detection rerun 已经补齐；
- 论文主表范围应当收敛到 4 个 viable categories；
- quantization 更适合当 deployment stress test，而不是唯一 novelty。

## 当前最重要的结论

1. **intervention works**
   strict Phase C 在去掉显式 `Unknown` 提示后仍然有效，说明效果不是靠答案 hint 撑起来的。

2. **`k=2` 略强于 `k=3`，但不是压倒性优势**
   `k=2` 在多个 timing 条件下都略优，但不适合写成特别强的机制 claim。

3. **timing 有价值，但 matched control 让故事变弱了一些**
   把 `random` 收紧成 `matched_random` 后，timing 和 control 的差距缩小；所以现在更安全的说法是“timing 可能有帮助”，而不是“timing 明显决定性更强”。

4. **hidden-state detection 仍然成立，但 transition-specificity 不应 oversell**
   sample-grouped held-out rerun 下，`hidden_only` 仍然强，但 matched control 也不弱，因此 transition-specificity 更像支持性证据，而不是决定性证据。

5. **主结果范围不该再追求全类别**
   当前最适合放进 paper 主表的是：
   - `age`
   - `disability_status`
   - `religion`
   - `ses`

## 当前最值得引用的结果

### 1. Phase C：biased-only timing 结果

最值得继续引用的是 `k=2` 主实验：

| 条件 | correction | remained_biased | avg_generated_tokens | 说明 |
|---|---:|---:|---:|---|
| `no_intervention` | 0.0000 | 1.0000 | 725.17 | 强基线 |
| `matched_random` | 0.8408 | 0.1551 | 366.89 | 收紧后的公平 control |
| `t-1` | 0.8571 | 0.1388 | 367.22 | 略高于 matched control |
| `t` | 0.8531 | 0.1469 | 372.07 | 略高于 matched control |
| `t+1` | **0.8612** | **0.1347** | 371.03 | 当前 `k=2` 主实验最强 |
| `end_of_reasoning` | 0.7661 | 0.2258 | 752.97 | 当前弱于 transition-aware timing |

补充观察：

- 扩展 timing 中，`k=2 t-2` 和 `k=2 t+1` 大体并列最强；
- `t+2` 没有继续提升；
- 因此更像是“第二个 transition 周围的一小段窗口有用”，而不是某个单点绝对最优。

### 2. sample-grouped detection rerun

这是 protocol cleanup 后更应该引用的 detection 结果。

| 设置 | held-out test AUROC |
|---|---:|
| `k=2 transition hidden_only` | `0.9020` |
| `k=2 matched_control hidden_only` | `0.8760` |
| `k=3 transition hidden_only` | `0.9113` |
| `k=3 matched_control hidden_only` | `0.8705` |
| `position_only` | `~0.69-0.70` |

当前更安全的解读：

- hidden-state detection 在 sample-grouped held-out 下仍然成立；
- `position_only` 只能算 weak-go；
- matched control 也不弱，所以不应再把 transition-specificity 写得过强；
- `position + hidden` 与 `hidden_only` 基本持平，不必强调组合特征有明显额外收益。
